command line

Run nanosite with no arguments to create a new site. If a site exists, run nanosite to start a development server, where you can preview your site live at http://localhost:8000/.

For more options:

  • nanosite build: Build the site without running the server.
  • nanosite serve: Run the server explicitly.
    • Use --port [XXXX] to set a custom port. Default: port 8000.
  • nanosite publish: Runs meta/publish: see publishing.
  • nanosite import [package-name]: Import a package: see packages.
    • Use --force to force import, even if package already installed.
    • Use nanosite --set-package-url [URL] to set the URL of the package repository (and save it as default).
  • nanosite clean: Empty the output directory.
  • nanosite delete: Delete the whole site.

Optional arguments:

  • -s [site-directory]: Use this as the directory of the site to build/serve/publish.
  • -o [output-directory]: Output to this directory when building/serving.
  • -m [meta-directory]: Use alternate meta directory.
