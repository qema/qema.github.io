Model¶

The objects used to represent a dataset.

class convokit.model.Corpus(filename=None, utterances=None, merge_lines=False, subdivide_users_by=[], delim=', ')¶

Represents a dataset, which can be loaded from a JSON file, CSV file or a list of utterances.

If a CSV file, the first row should consist of key names. Unknown key names will have their attributes added to the “user-info” entry.

Parameters:
  • filename – path of json or csv file to load
  • utterances – list of utterances to load
  • merge_lines – whether to merge adjacent lines from the same user if the two utterances have the same root.
  • subdivide_users_by – collection of strings corresponding to attribute names defined in the “user-info” entry. Use this if you want to count the same user as being different depending on attributes other than username. For example, in the Supreme Court dataset, users are annotated with the current case id. You could use this to count the same person across different cases as being different users.
  • delim – if loading a csv, specifies the delimiter string.
Variables:

utterances – dictionary of utterances in the dataset, indexed by id.
pairwise_exchanges(selector=None, user_names_only=False)¶

Get all directed pairwise exchanges in the dataset.

Parameters:
  • selector – optional function that takes in a speaker user and a replied-to user and returns True to include the pair in the result, or False otherwise.
  • user_names_only (bool) – if True, index conversations by user names rather than user objects.
Returns:

Dictionary mapping (speaker, target) tuples to a list of utterances given by the speaker in reply to the target.
speaking_pairs(selector=None, user_names_only=False)¶
Get all directed speaking pairs (a, b) of users such that a replies
to b at least once in the dataset.
Parameters:
  • selector – optional function that takes in a speaker user and a replied-to user and returns True to include the pair in the result, or False otherwise.
  • user_names_only (bool) – if True, return just pairs of user names rather than user objects.
Returns:

Set containing all speaking pairs selected by the selector function, or all speaking pairs in the dataset if no selector function was used.
subdivide_users_by(attribs)¶

Use this if you want to count the same user as being different depending on attributes other than username. For example, in the Supreme Court dataset, users are annotated with the current case id. You could use this to count the same person across different cases as being different users.

Repeated calls to this method will override previous subdivisions.

Parameters:attribs (Collection) – Collection of attribute names to subdivide users on.

class convokit.model.User(name=None, info={})¶

Represents a single user in a dataset.

Parameters:
  • name (str) – name of the user.
  • info (dict) – arbitrary dictionary of attributes associated with the user.
Variables:
  • name – name of the user.
  • info – dictionary of attributes associated with the user.
identify_by_attribs(attribs)¶

Identify a user by a list of attributes. Sets which user info attributes should distinguish users of the same name in equality tests. For example, in the Supreme Court dataset, users are labeled with the current case id. Call this method with attribs = [“case”] to count the same person across different cases as different users.

Parameters:attribs (Collection) – Collection of attribute names.
class convokit.model.Utterance(id=None, user=None, root=None, reply_to=None, timestamp=None, text=None)¶

Represents a single utterance in the dataset.

Parameters:
  • id – the unique id of the utterance. Can be any hashable type.
  • user – the user giving the utterance.
  • root – the id of the root utterance of the conversation.
  • reply_to – id of the utterance this was a reply to.
  • timestamp – timestamp of the utterance. Can be any comparable type.
  • text (str) – text of the utterance.
Variables:
  • id – the unique id of the utterance.
  • user – the user giving the utterance.
  • root – the id of the root utterance of the conversation.
  • reply_to – id of the utterance this was a reply to.
  • timestamp – timestamp of the utterance.
  • text – text of the utterance.
